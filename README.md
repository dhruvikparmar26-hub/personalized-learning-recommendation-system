# LearnFlow — Personalized Learning Recommendation System

A full-stack, **real-time** learning recommendation platform featuring TF-IDF content-based filtering with live re-ranking, Claude AI streaming chat, WebSocket connections, and Server-Sent Events for instant UI updates.

## 🏗️ Architecture

```
┌─────────────────────┐     ┌──────────────────────┐     ┌─────────────────┐
│   React Frontend    │────▶│   FastAPI Backend     │────▶│   ML Engine     │
│  WebSocket + SSE    │◀────│  REST + WS + SSE     │◀────│  TF-IDF + Sim   │
└─────────────────────┘     └──────────┬───────────┘     └─────────────────┘
                                       │
                            ┌──────────┼───────────┐
                            ▼          ▼           ▼
                      ┌──────────┐ ┌────────┐ ┌──────────┐
                      │  Claude  │ │ Redis  │ │PostgreSQL│
                      │ Streaming│ │Pub/Sub │ │ Supabase │
                      └──────────┘ └────────┘ └──────────┘
```

### What makes this genuinely real-time:

| Feature | Protocol | Purpose |
|---------|----------|---------|
| **AI Chat** | WebSocket | Persistent bidirectional connection for streaming Claude responses |
| **Live Re-ranking** | Server-Sent Events | Recommendations reorder instantly when user gives feedback |
| **Claude Streaming** | SSE | Explanations appear word-by-word as they're generated |

### The smart engineering choice: Re-ranking > Retraining

Full model retraining takes **minutes**. Re-ranking a scored list takes **milliseconds**.

```
final_score = similarity_score × topic_weight × recency_factor
```

- `similarity_score`: Cosine similarity from TF-IDF [0, 1]
- `topic_weight`: Adjusted by feedback — like (+0.2), skip (-0.3), clamped to [0.1, 3.0]
- `recency_factor`: Time decay — `1 / (1 + days × 0.1)`

## 🚀 Quick Start

### Prerequisites
- Python 3.11+
- Node.js 20+
- Docker & Docker Compose (recommended)

### Option 1: Docker (Recommended)

```bash
# Clone
git clone https://github.com/yourusername/learnflow.git
cd learnflow

# Configure
cp backend/.env.example backend/.env
# Edit backend/.env with your keys

# Run everything
docker-compose up --build
```

Frontend: http://localhost:5173 | Backend: http://localhost:8000/docs

### Option 2: Manual Setup

```bash
# Backend
cd backend
python -m venv venv
venv\Scripts\activate        # Windows
pip install -r requirements.txt
python -m scripts.precompute  # Build TF-IDF model
uvicorn src.api.main:app --reload

# Frontend (new terminal)
cd frontend
npm install
npm run dev
```

## 🔌 API Endpoints

| Method | Endpoint | Type | Description |
|--------|----------|------|-------------|
| POST | `/api/users` | REST | Register user |
| POST | `/api/onboarding/` | REST | Submit quiz answers |
| GET | `/api/recommendations/{id}` | REST | Get ranked courses |
| POST | `/api/feedback/` | REST | Like/skip/save/complete |
| GET | `/api/stream/recommendations/{id}` | SSE | Live recommendation updates |
| GET | `/api/stream/explanation/{id}` | SSE | Claude streaming explanation |
| WS | `/api/chat/{id}` | WebSocket | AI chat assistant |
| GET | `/api/health` | REST | Health check |

Full interactive docs at `/docs` (Swagger UI).

## 🛠️ Tech Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| Frontend | React + Vite | Fast dev server, modern tooling |
| Styling | Vanilla CSS | Full control, no framework lock-in |
| Animation | Framer Motion | Layout animations for card reordering |
| Backend | FastAPI | Native async, WebSocket support, auto-docs |
| ML | scikit-learn | TF-IDF + cosine similarity, fast and explainable |
| AI | Claude API | Streaming responses, context-aware |
| Cache/Events | Redis | Pub/sub for events, sub-ms reads for scores |
| Database | PostgreSQL | Reliable, full-featured, Supabase hosted |
| Deployment | Render + Vercel | Free tier, WebSocket support |
